# Stage 6, Assembly Reference (v2.2)

## Purpose

Stage 6 is production, not thinking. The thinking happened in stages 1 to 5. Stage 6 takes the artifacts and builds six final deliverables:

1. **Premium proposal deck** (PDF), built via `premium-deck-strategist` skill, Deep Blue design language, Rule of Three methodology
2. `budget.xlsx`, the detailed Excel budget with formulas
3. `gamma-prompt.md`, the ready-to-paste prompt for Gamma.app (interactive companion version)
4. `summary.md`, a 1-page executive summary for intro messages
5. `kpis-scorecard.md`, a standalone KPI measurement plan (also embedded in the deck)
6. Trello debrief card, created in the "Bold, Debriefs" board, due the day after the event

**v2.2 change:** The PDF is no longer generated via the generic `pdf` skill. It is generated by `premium-deck-strategist`, which produces a polished slide-by-slide deck with Bold's quality standard: punchy titles, visual layout specs, on-slide text max 5 words per bullet, and speaker notes.

Order: XLSX first (numbers lock), then premium deck (tells the story with numbers confirmed), then Gamma prompt (digital companion), then KPIs scorecard, then summary, then Trello card.

## 2. Building the premium deck (proposal.pdf) [v2.2, new method]

### Why premium-deck-strategist
Bold's proposals compete on design quality. A markdown-to-PDF pipeline produces documents. A deck strategist produces presentations. Clients read the PDF like a pitch, not a contract.

### Invocation pattern

At the start of Stage 6 deck generation, Claude invokes `premium-deck-strategist` with this brief:

```
Build a premium proposal deck for [Client Name] for the event "[Event Name]".
Audience: [from brief field 6, primary audience description, plus "this is the decision-maker at the client"].
Source material:
  - Brief: proposals/<slug>/01-intake/brief.md
  - Strategic reading: proposals/<slug>/01-intake/challenges.md
  - Brand heart: proposals/<slug>/03-brand-heart/brand-system.md
  - Mood direction: proposals/<slug>/04-specialists/visual/mood-direction.md
  - Mockups: proposals/<slug>/04-specialists/visual/mockups/*.png
  - Agenda: proposals/<slug>/04-specialists/content/agenda.md
  - Operations: proposals/<slug>/04-specialists/operations/logistics.md
  - Menu: proposals/<slug>/04-specialists/culinary/menu.md
  - Budget summary (from budget.json): ₪[total] + שורות מותנות [list]
  - KPIs: proposals/<slug>/01-intake/brief.md field 16

Required slide order (non-negotiable):
  1. Cover (event name + client + date + Bold watermark)
  2. About this event (one-line reason-why from brief field 1)
  3. Strategic reading (3 key insights from challenges.md)
  4. The creative concept (brand-system.md fields 1-3, hero mockup)
  5-9. Visualization (5 mockups: arrival, main space, hero moment, catering, details)
  10. The experience (agenda highlights, 3 anchor moments)
  11. Operations (run-sheet outline, 1 slide)
  12. Culinary (menu narrative, 1 slide)
  13. Investment (budget summary, clear price + conditional lines)
  14. KPIs and success measurement (table from brief field 16, closing line about 24h debrief)
  15. Timeline (delivery dates + decision points)
  16. Next steps (clear CTA, contact info)

Design directive: the visual system of this specific deck must pull from brand-system.md (palette, typography, materials), NOT from premium-deck-strategist's default Deep Blue. Deep Blue is the fallback ONLY if brand-system.md is incomplete.

Voice rules (non-negotiable):
  - Hebrew-first, all client-facing slides in Hebrew
  - No em-dash, no en-dash
  - No clichés from the banned list in SKILL.md
  - Specific numbers not adjectives
  - Max 5 words per bullet on-slide; depth goes in speaker notes
  - No Bold logos or credits on client-facing slides unless the client asked for them

Output: proposals/<slug>/06-assembly/proposal.pdf
```

### What premium-deck-strategist returns
- A structured PDF with the 16-slide order above
- Each slide has: punchy title (max 7 words), visual layout spec honored, on-slide text under word limit, speaker notes 2-4 sentences
- Brand system from Stage 3 is applied (palette, typography, materials cues)
- Mockups from Stage 4a embedded in the right slots (slides 4-9)

### Post-generation QA
- Check slide count matches the 16-slide spec
- Check brand palette matches brand-system.md exactly
- Check all mockups render correctly (no missing images)
- Check Hebrew RTL rendering is correct on all text-heavy slides
- Check investment slide total matches budget.xlsx total

If any check fails, Claude asks premium-deck-strategist to regenerate the specific slide, not the whole deck.

### Budget threshold fallback
For proposals under ₪25K, premium-deck-strategist is overkill. Fall back to a simpler 8-slide version. Claude makes this decision automatically based on budget.json subtotal.

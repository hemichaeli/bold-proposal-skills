# Design Passes — pipeline (deck)

Optional visual-refinement layer for `premium-deck-strategist`. Apply after the deck narrative + structure exist. All passes are opt-in; run only those whose triggers match the request.

## Order (fixed)

1. **reference-enrichment** — taste / art direction (before or alongside strategy)
2. **strategist** — narrative + slide structure (this skill's core job)
3. **layout-cleanup** — normalize slide layout / type / alignment
4. **motion** — consistent transitions + purposeful builds

## Override rule

Clarity, legibility, and presenter experience win over any visual flourish. A pass step that would hurt readability, make the room wait on an animation, or bury the message is downscoped or dropped.

## Selection

| Pass | Run when the request implies |
|---|---|
| reference-enrichment | premium, less generic, stronger design, better taste, modern look |
| layout-cleanup | cleaner slides, fix layout, too much text, tighten, more professional |
| motion | transitions, animate, builds, polished feel |

No design signal + content-only request → run zero passes.

## What each pass produces

- reference-enrichment → art-direction note + enriched deck brief (mood, type, palette discipline)
- layout-cleanup → per-slide layout fixes (findings → fix)
- motion → one deck-wide transition + a short list of comprehension-serving builds

## Boundary

Passes refine visual treatment only. They never rewrite the narrative, reorder slides, or change the argument — that is the strategist's domain.

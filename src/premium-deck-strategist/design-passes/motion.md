# Pass - motion (deck)

Slide transitions and builds, tastefully. Runs last, on the cleaned deck.
Subtle by default; motion serves comprehension, never decoration.

## Apply when
"transitions", "animate", "builds", "make it feel polished/alive", premium feel.

## Do
- **One consistent transition** for the whole deck - a gentle fade or a soft
  push. Same transition everywhere reads as craft.
- **Builds only where they help** - reveal a list item as you speak to it,
  animate a chart/metric in, step through a process diagram. The build should
  track the spoken narrative.
- Keep transitions **short** - never make the presenter wait on an animation.
- Respect the presentation tool's **reduce-motion / auto-advance** settings.

## Don't (motion spam)
- no spin, bounce, fly-in, swivel, typewriter, or per-slide different transitions
- no animation on every element "because we can"
- no transition so long it interrupts the talk track
- never rely on motion to convey meaning that isn't also in the static slide

## Hard boundary
Subtle + consistent. If in doubt, less motion. A deck that reads perfectly with
all animation stripped is the baseline; motion only adds polish on top.

## Output
One deck-wide transition choice + a short list of slides that get a
comprehension-serving build, with the build described per slide.
